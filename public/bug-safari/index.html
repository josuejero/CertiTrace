Requirement CVT-VER-01 Severity: Major

Verification modal closes before capture

Requesting a verification letter from a record detail page sometimes dismisses the modal without logging the audit entry.

Reproduction steps
  • Search for "Navarro" and open the first result.
  • Click "Request verification" and choose the default letter type.
  • Immediately fill the contact fields and submit.
  • Observe the modal disappear without confirmation and no audit log entry.

Root cause: The submit handler optionally short-circuits when maintenance-mode flags are stale, leaving the modal in a pending state.

PR #1: Fix verification modal lifecycle

Requirement NFR-ACC-01 Severity: Minor

Keyboard flow skips maintenance banner

Tabbing through the search page ignores the maintenance banner when the config switches to maintenance mode.

Reproduction steps
  • Toggle `maintenance.json` to `maintenanceMode: true` locally (or use the dev control).
  • Load the search page and press Tab repeatedly.
  • Notice the focus jumps from the header directly to the search form, skipping the banner.

Root cause: The banner renders outside the sequential container, so `tabindex` reorder doesn’t capture it when maintenance state flips dynamically.

PR #2: Reset banner focus order

Requirement CVT-MNT-01 Severity: Major

Maintenance banner state disconnect

Turning maintenance on via `maintenance.json` doesn’t always disable the verification request button unless the page is refreshed.

Reproduction steps
  • Start with maintenance mode off and load the app.
  • Open the console and toggle the config file locally (or flip the control panel).
  • Click “Request verification” and notice the button remains enabled despite the banner.

Root cause: The config is read once on startup; the button state is not data bound to the banner’s derived flag, so they drift.

PR #3: Bind banner to request control
